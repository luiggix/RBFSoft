TUNA::RBF Documentation

TUNA::RBF is a set of template functions, classes and namespaces for numerically solving Partial Differential Equations using Radial Basis Functions Mesh-free methods. It also can be used to interpolate scattered data. Currently this library contains:

  • Kernels (1D, 2D and 3D)
    • Multiquadric (MQ)
    • Thin-Plate Splines (TPS)
  • Solvers for linear systems
    • Gauss elimination
    • GMRES
  • Preconditioners
    • Jacobi
    • Approximated Cardinal Basis Functions
  • Classes for point generation in different shapes
    • Rectangle
    • Semicircles
    • Boxes
    • Sub-domains for DD (rectangles)
  • KDTree algorithm from libkdtree++.

Quick instructions for the impatient

  • Install BLAS and LAPACK
  • Install FLENS
  • Unpack TUNA::RBF
  • Go to the examples directory.
    • Modify rules.in and rulesMPI.in files.
    • compile and execute each example to get used to the components of TUNA::RBF.
    • For more info about the examples click on the Files tab of the above menu.

Requirements

  • FLENS: Flexible Library for Efficient Numerical Solutions. The components of this library are the building blocks of TUNA::RBF, so you need to install and test it before start using TUNA::RBF. FLENS is a convenient and efficient C++ interface for BLAS and LAPACK, in such a way that is a very useful tool for the creation of serious scientific software in C++. FLENS requires a working BLAS and LAPACK implementation on your system, and the CBLAS interface for BLAS. There exist different BLAS and LAPACK implementations, the two tested and recommended so far are:
    • ATLAS and LAPACK,
    • Intel Math Kernel Library (MKL).
  • libkdtree++ is an STL-like C++ template container implementation of k-dimensional space sorting, using a kd-tree. The source of this library is included with TUNA::RBF, so you do not need to install it at all.

Installation

TUNA::RBF source code is completely in the headers, so the only libraries you need to compile and to install are FLENS and a flavor of BLAS and LAPACK. Once installed the requirements, all you need to do is to unpack TUNA::RBF and to define the next environment variables according to your system. These variables are in the files examples/rules.in and examples/rulesMPI.in (modify both of them).
  • FLENS: path to the FLENS library, as installed in your system.
  • ATLAS: path to the ATLAS library, as installed in your system.
  • RBF: path where the TUNA::RBF library will be installed (unpacked).
Now go to the examples subdirectory and compile (make) and execute each example. Please read the documentation of every example (click on the Files tab of the html documentation for a complete explanation of the examples).

Compiler Options

  • -DNDEBUG : used for FLENS to avoid bounds check and other debugging stuff. When used the performance is drastically improved.
  • -DWITH_GNUPLOT : to launch GNUplot during the execution of the examples.
  • -D_1D_ : for 1D problems (could be deprecated in the future).
  • -D_2D_ : for 2D problems (could be deprecated in the future).
  • -D_3D_ : for 3D problems (could be deprecated in the future).
  • -DMPICH_IGNORE_CXX_SEEK : for compiling MPI-C++ bindings.
  • -DVECLIB : for linking with BLAS and LAPACK in MacOS.

Post-procesing

The results of the examples are written in ASCII format and can be plotted directly using GNUplot. If you are using -DWITH_GNUPLOT in your compiling flags, then gnuplot is launched automatically during the execution of every example. To use this feature be sure you have installed GNUplot in your system.
